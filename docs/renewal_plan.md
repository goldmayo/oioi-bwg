### 메인 화면 리뉴얼 계획

방문자의 75%가 모바일 사용자이므로 모바일 사용자 UX 개선을 위해서 모바일 사이즈 친화 UX 고려

#### ASIS

- 홈: 앨범 목록
- 응원법:
  - 앨범 페이지의 곡 아이템을 클릭하면 응원법 페이지로 이동
  - 응원법 종류가 오피셜, 비공식(기본)으로 구분 없이 적용되어 있음.

- 검색: 없음
- 하단 네비게이션바: 없음

#### TOBE

- 홈:
  - 현재 화면에서 스크롤을 많이 하고 있다.
  - 스크롤이 긴 이유: 500x500 사이즈의 앨범 커버가 세로 스택으로 쌓여 있어 스크롤 영역이 길어짐.
  - 디스코그래피 목록으로 리스트 아이템화 한다.
    - 정사각형 이미지 / 앨범명 + 발매일 + 트랙 수 조합.
    - 가장 최신 앨범이 위로 오도록 정렬.
    - 디스코 그래피 목록(앨범)을 클릭시 앨범 상세 페이지로 이동.

  - "응원법 연습하기" 버튼으로 응원법 페이지로 이동.

- 응원법 상세 페이지
  - 응원법 종류를 뱃지로 표현 (오피셜, 비공식)

## 기술적 제안 (Technical Strategy)

### 1. 홈 화면 (디스코그래피 리스트 뷰)

- 단순히 세로로 나열하는 것을 넘어, Spotify/멜론 같은 음원 앱 스타일의 깔끔한 ListItem (왼쪽 썸네일, 오른쪽 텍스트 조합)으로 구현합니다.
- 썸네일 이미지는 Next.js의 `Image` 컴포넌트를 이용해 모바일 사이즈에 맞게 동적 리사이징(`sizes="(max-width: 768px) 100vw, 33vw"`)을 적용하여 로딩 속도를 향상시킵니다.

### 2. 응원법 상세 페이지 공유하기

- 접속자의 절대다수가 모바일 유저이므로, Web Share API (`navigator.share()`)를 사용하여 OS 네이티브 공유 다이얼로그(카카오톡, 메시지 등)가 뜨도록 연결합니다.
- 데스크탑처럼 Web Share API가 제한적인 기기를 위한 대체재(Fallback) 기능으로 화면에 '클립보드에 주소가 복사되었습니다.'라는 Toast 알림 표시를 구현합니다.
- 토글된 뱃지 상태(예: 비공식만 보기 등)를 URL 쿼리 파라미터(현재 설치된 `nuqs` 적극 활용)와 동기화하여, 공유받은 사람도 동일한 옵션 값을 볼 수 있도록 설계합니다.

### 3. 응원법 리스트 & 검색 최적화

- 곡과 앨범 수는 한정적이므로 매 검색 시 DB에 요청하여 지연시간을 만드는 대신, 데이터 패치를 통해 초기 렌더링(SSG/상태 보관) 후 클라이언트 사이드에서 즉각적으로 필터링을 수행하는 'Instant Search'를 구현하여 답답함 없는 경험을 제공합니다.

### 4. 하단 네비게이션 / 좌측 사이드바 구조 하이브리드화

- Next.js의 `RootLayout`을 이용해 전역 레이아웃 요소로 배치합니다.
- CSS 미디어 쿼리(Tailwind의 `md:` 등)를 활용하여 폭이 좁은 모바일에서는 하단 탭(`fixed bottom-0 w-full`)으로, 일정 폭 이상 넓은 PC 화면에서는 좌측 탭(`fixed left-0 h-screen w-64`)으로 부드럽게 전환(Responsive Desktop Layout)되는 것을 기본 골조로 삼습니다.

### 5. 인앱 브라우저(In-App Browser) 탈출 로직

- 카카오톡, 인스타그램 등 앱 내장 브라우저(웹뷰) 접속 시 발생하는 세션 유실 및 Web Share API 동작 오류를 방지하기 위해 사용자 진입 시 기본 브라우저(Chrome, Safari 등)로 강제 우회시킵니다.
- **적용 로직**: `sessionStorage`를 이용해 무한 리다이렉트 루프를 방지하고, 카카오톡 전용 스킴(`kakaotalk://web/openExternal`) 및 안드로이드 크롬 인텐트(`intent://`)를 활용한 네이티브 우회 로직을 구성합니다.
- **적용 위치**: 공유된 링크를 타고 어느 페이지로 들어오든 앱 시작 시 즉각 탈출할 수 있도록, `Next.js`의 최상단 루트 레이아웃(`src/app/layout.tsx`) 최상부에 `use client` 기반 가드 컴포넌트(`<InAppBrowserGuard />`)로 마운트합니다.
